✓ Safe — local aggregation + lock
Each thread keeps its own localMin / localMax
No shared state during loop ✓
At the end, each thread calls localFinally
Only 1 lock per thread (not per element!) ✓
Strategy: partition → local reduce → merge
• localInit() → each thread starts (MaxValue, MinValue)
• body() → update local only — no lock needed
• localFinally() → lock once, merge into global
Lock count = #threads (e.g. 4), not array.Length (e.g. 10M) → near-zero contention.
